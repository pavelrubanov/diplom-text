\section{Описание решения}
\label{sec:solution}

\subsection{Требования к системе}

Требования сформулированы совместно с консультантом (м.н.с.\ лаборатории Гербарий ГБС РАН, Шкурко~А.~В.)
на основе технического задания.

\subsubsection*{Требования заказчика}

\begin{enumerate}
    \item Автоматическое распознавание формы листа и построение полигональной маски контура.
    \item Определение длины и ширины полигона.
    \item Расстановка точек вдоль перпендикуляров к главной оси с заданным шагом (например, $\frac{1}{2}$ или $\frac{1}{4}$ длины листа).
    \item Просмотр исходного изображения и маски рядом на экране.
    \item Возможность задать масштабный коэффициент ($1~\text{пиксель} = \ldots~\text{мм}$) для вывода измерений в физических единицах.
    \item Экспорт результатов в табличный формат (CSV / Excel).
    \item Пакетная обработка серий изображений.
    \item Возможность ручной корректировки автоматически построенной маски.
    \item Полностью автоматический режим обработки изображения: построение масок всех объектов без участия пользователя, автоматический отбор корректных листьев и запуск измерений.
\end{enumerate}

\subsubsection*{Функциональные требования}

На основе технического задания выделены следующие варианты использования:

\begin{itemize}
    \item \textbf{UC1.} Загрузить набор изображений для обработки.
    \item \textbf{UC2.} Получить маску сегментации по интерактивным подсказкам (точки, ограничивающий прямоугольник).
    \item \textbf{UC3.} Скорректировать маску вручную в режиме кисти/ластика.
    \item \textbf{UC4.} Добавить подтверждённую маску в буфер или удалить её.
    \item \textbf{UC5.} Задать масштабный коэффициент пиксель~→~мм.
    \item \textbf{UC6.} Запустить пакетное измерение всех сохранённых масок.
    \item \textbf{UC7.} Экспортировать таблицу результатов в~CSV.
    \item \textbf{UC8.} Выбрать тип листа для автоматической обработки (из списка типов, для которых обучена модель отбора).
    \item \textbf{UC9.} Запустить полностью автоматический режим: система строит маски всех объектов на изображении, отбрасывает некорректные с помощью обученного классификатора и добавляет оставшиеся в буфер без участия пользователя.
    \item \textbf{UC10.} Просмотреть результаты автоматического отбора (корректные и отброшенные маски с их классами) и при необходимости скорректировать отдельные маски вручную (переход в~UC3).
\end{itemize}

\subsubsection*{Нефункциональные требования}

\begin{itemize}
    \item Работоспособность на стандартных рабочих станциях без выделенного GPU (инференс всех моделей на CPU). При наличии GPU приложение должно автоматически задействовать его для ускорения без дополнительной настройки со стороны пользователя.
    \item Время отклика модели сегментации не более 3~с на одно изображение в интерактивном режиме.
    \item Поддержка форматов JPEG, PNG, TIFF.
    \item Воспроизводимость результатов: одни и те же входные данные и подсказки дают одинаковый вывод.
    \item Единый формат хранения масок (PNG) и измерений (CSV) для последующей статистической обработки.
    \item Расширяемость автоматического режима: добавление поддержки нового типа листа сводится к обучению классификатора отбора на соответствующей размеченной выборке, без изменений в архитектуре приложения.
\end{itemize}

\subsection{Архитектура приложения}

Приложение реализовано на языке Python с использованием фреймворка PyQt6 для графического интерфейса
и библиотеки PyTorch для инференса модели MobileSAM.
Исходный код опубликован по адресу \url{https://github.com/pavelrubanov/segmentation_system}.

Система состоит из двух слоёв: \textit{ядра} (core) и \textit{пользовательского интерфейса} (ui).
Ядро не зависит от GUI и включает обёртку над моделью сегментации, модель данных изображений
и модуль измерений. Слой UI реализует окна и виджеты PyQt6.

\subsection{Диаграмма классов}

Ключевые классы системы:

\begin{itemize}
    \item \texttt{Predictor}~--- обёртка над MobileSAM.
          Метод \texttt{predict(points, labels, box)} принимает список координат точек,
          их меток ($+1$ / $-1$) и опциональный ограничивающий прямоугольник;
          возвращает бинарную маску в виде массива \texttt{uint8} (значения 0 и 255).
          Инференс выполняется на CPU с помощью \texttt{torch.no\_grad()}.

    \item \texttt{ImageWithMasks}~--- датакласс, хранящий изображение (\texttt{numpy} RGB) и имя файла.
          Маски не хранятся в памяти~--- только пути к файлам на диске, что снижает потребление RAM
          при пакетной обработке.

    \item \texttt{LeafMetrics}~--- результат измерения одного листа:
          \texttt{length\_px}, \texttt{width\_px} и список \texttt{SectionWidth}
          для семи поперечных сечений.

    \item \texttt{SegmentationCanvas}~--- виджет \texttt{QGraphicsView} с двумя режимами:
          \textit{PROMPTS} (клики мышью задают подсказки для SAM) и
          \textit{EDIT} (кисть/ластик для ручной коррекции маски).
          Поддерживает зум в точку курсора и панорамирование по \texttt{Ctrl+ЛКМ}.

    \item \texttt{MasksBuffer}~--- галерея миниатюр сохранённых масок (256$\times$256 px).
          Маски индексируются по имени изображения (\texttt{image\_mask\_0001.png} и т.д.).
\end{itemize}

\subsection{Диаграмма последовательности}

Типовой сценарий работы пользователя:

\begin{enumerate}
    \item Загрузить набор изображений~→ \texttt{NavigationWindow} открывает \texttt{SegmentationWindow}.
    \item Щёлкнуть по листу левой кнопкой мыши (positive point)~→
          \texttt{SegmentationCanvas} вызывает \texttt{Predictor.predict()}~→
          полупрозрачный overlay маски отображается поверх изображения.
    \item При необходимости переключиться в режим EDIT и подправить маску кистью.
    \item Подтвердить маску~→ файл PNG сохраняется на диск, \texttt{MasksBuffer} обновляет миниатюру.
    \item Перейти в \texttt{ProcessingWindow}, выбрать папку с масками, запустить измерения.
    \item \texttt{leaf\_measure.measure\_leaf()} обрабатывает каждую маску~→ CSV экспортируется.
\end{enumerate}

\subsection{Алгоритм параметрических измерений}

Модуль \texttt{leaf\_measure.py} реализует измерение геометрических характеристик листа
на основе анализа главных компонент (PCA) контура маски.

\begin{enumerate}
    \item Из бинарной маски извлекается внешний контур (OpenCV \texttt{findContours}).
    \item К координатам точек контура применяется PCA:
          первый главный вектор соответствует главной оси листа (длина),
          второй~--- перпендикулярному направлению (ширина).
    \item Длина листа вычисляется как проекция контура на первый главный вектор.
    \item Ширина в точке $t \in \{0.125, 0.25, 0.375, 0.5, 0.625, 0.75, 0.875\}$~---
          расстояние между двумя пересечениями перпендикуляра к главной оси с контуром.
    \item При наличии масштабного коэффициента значения пересчитываются в миллиметры.
\end{enumerate}

Для каждой обработанной маски генерируется изображение-визуализация (\texttt{*\_vis.png})
с нанесёнными контуром, осями и поперечными сечениями.

\subsection{Полностью автоматический режим}
\label{sec:auto_mode}

Помимо интерактивного сценария (раздел~\ref{sec:solution}), система поддерживает
полностью автоматический режим, в котором участие эксперта не требуется.
В этом режиме для исходного изображения выполняется автоматическая сегментация
всех объектов (SAM Automatic Mask Generator), после чего полученный набор
кандидатов передаётся в модуль отбора масок. Только объекты, классифицированные
как корректные листья, участвуют в последующих параметрических измерениях.

Такая постановка задачи отличается от интерактивной тем, что невозможно
полагаться на человеческую верификацию каждой маски. Ошибка модели
автоматически попадает в выходную таблицу и влияет на результат исследования.
Поэтому к модулю отбора масок предъявляется более жёсткое требование:
минимизация доли некорректных листьев, попавших в итоговую выборку.

Автоматический и полуавтоматический (интерактивный) режимы сосуществуют
в приложении и решают разные задачи:

\begin{itemize}
    \item \textbf{Интерактивный режим} (раздел~\ref{sec:solution},
          диаграмма активностей на рис.~\ref{fig:activity})~---
          пользователь указывает объект подсказкой (точкой или bounding box),
          а модель SAM строит маску по конкретному запросу.
          Этот режим остаётся основным по двум причинам:
          во-первых, одиночный запрос к SAM выполняется заметно быстрее
          полной автоматической сегментации всего изображения
          (используется SAM AMG), что критично при работе биолога
          с десятками снимков;
          во-вторых, он применим к произвольному типу листа, даже если
          для него ещё не обучен классификатор отбора~---
          в частности, именно с помощью интерактивного режима была
          собрана исходная размеченная выборка для обучения
          HybridLeafClassifier.
    \item \textbf{Автоматический режим} применяется, когда для текущего
          типа листа доступна обученная модель отбора масок и требуется
          массовая обработка серии изображений без участия эксперта.
\end{itemize}

\subsubsection*{Два подхода к отбору масок}

Рассмотрены и реализованы два принципиально разных подхода.

\begin{enumerate}
    \item \textbf{Supervised-классификация} на три класса
          (\textit{хороший лист}, \textit{плохой лист}, \textit{не лист}).
          Требует размеченной выборки всех трёх классов, но позволяет явно
          различать типы ошибок: плохие листья~--- это объекты листовой природы
          с дефектами (перекрытия, рваные края), а не-листья~--- посторонние
          объекты (фон, тени, артефакты сегментации). В приложении эти два
          класса обрабатываются по-разному.
    \item \textbf{One-class anomaly detection}. Модель обучается только
          на выборке корректных листьев и выдаёт числовой score аномальности;
          объекты со score выше порога отбрасываются. Не требует разметки
          плохих примеров, но не различает типы отклонений.
\end{enumerate}

Архитектура и результаты supervised-варианта описаны в
подразделе~\ref{sec:hybrid_clf}, one-class моделей~---
в подразделе~\ref{sec:filters}. Обе группы моделей используют
общий набор признаков и общую стратегию аугментации, поэтому
техника аугментации данных вынесена в отдельный
подраздел~\ref{sec:augmentation}.

\subsection{Аугментация данных}
\label{sec:augmentation}

Размеченная выборка для обучения моделей отбора небольшая (сотни
примеров на тип листа), поэтому обе группы моделей (supervised и
one-class) используют общую стратегию оффлайн-аугментации.
При загрузке каждый обучающий пример размножается в $K$ копий
(оригинал и $K-1$ трансформированных), копии собираются в
фиксированный тензор и предзагружаются в GPU, что устраняет
I/O на итерациях обучения. По умолчанию $K = 5$.

К аугментированным копиям применяются случайные трансформации:

\begin{itemize}
    \item \textbf{геометрические:} горизонтальное и вертикальное
          отражение; поворот на произвольный угол $\pm 45^{\circ}$;
    \item \textbf{цветовые:} изменение яркости и контраста
          (для HybridLeafClassifier диапазон $[0{,}7;\, 1{,}3]$,
          для one-class AE~--- более узкий $\pm 5\%$, чтобы не
          размывать цветовой признак, по которому разделяются
          \texttt{good\_leaf} и \texttt{non\_leaf}); для
          HybridLeafClassifier дополнительно~--- вариация насыщенности.
\end{itemize}

Геометрические трансформации применяются синхронно к изображению
и маске; после этого признаки, вычисляемые из маски (геометрические
и цветовые~--- см.\ подраздел~\ref{sec:hybrid_clf_features}),
пересчитываются из трансформированной пары. Это обеспечивает
согласованность между визуальным представлением и числовыми
дескрипторами каждой копии.

\subsection{Модель классификации: HybridLeafClassifier}
\label{sec:hybrid_clf}

\subsubsection*{Постановка задачи}

Для каждой маски-кандидата, полученной от SAM, требуется определить
класс объекта:

\begin{itemize}
    \item \texttt{good\_leaf}~--- корректный одиночный лист, пригодный
          для морфометрических измерений;
    \item \texttt{bad\_leaf}~--- лист с дефектами
          (перекрытия с другими объектами, неполная форма, рваные края);
    \item \texttt{non\_leaf}~--- объект нелистовой природы
          (фон, тень, артефакт сегментации).
\end{itemize}

Входные данные классификатора~--- пара ``фрагмент RGB-изображения,
вырезанный по ограничивающему прямоугольнику маски''
и ``бинарная маска того же объекта''. Выход~--- логиты по трём классам.

на CPU занимает менее 10~мс.

\subsubsection*{Обоснование архитектуры}

Выбор late fusion объясняется различной природой двух источников информации:

\begin{itemize}
    \item Пиксели фрагмента несут \textit{визуальную} информацию: цвет, текстуру,
          наличие окклюзий. Эта информация критична для отделения
          \texttt{non\_leaf} (фон, тени) от объектов листовой природы,
          но её частично можно выразить простыми статистиками
          (средние и std по каналам, медиана HSV) и передать в MLP-ветвь
          вместо того, чтобы CNN извлекала их с нуля.
    \item Бинарная маска несёт \textit{геометрическую} информацию о форме
          объекта. Такие показатели, как solidity и circularity,
          напрямую отличают гладкие одиночные листья от объектов
          с рваными краями или окклюзиями.
\end{itemize}

Handcrafted-признаки освобождают модель от необходимости извлекать их
самостоятельно, что существенно при малом размере обучающей выборки.
Hu-моменты обеспечивают инвариантность представления формы к сдвигу,
масштабу и повороту.

\subsection{Модели one-class}
\label{sec:filters}

Помимо supervised-классификации (HybridLeafClassifier,
подраздел~\ref{sec:hybrid_clf}), рассматривались one-class методы,
не требующие размеченной выборки плохих примеров. Обучение проводится
только на корректных листьях; модель выдаёт числовой score
аномальности, объекты со score выше порога отбрасываются.

\subsubsection*{Модели}

Все модели работают с RGB-фрагментом $128 \times 128$ с обнулённым
фоном (формат \texttt{\_img\_crop\_*.png}, как их сохраняет приложение).

\begin{itemize}
    \item \textbf{AE.} Свёрточный автоэнкодер с 5~блоками
          \texttt{Conv}$\to$\texttt{BN}$\to$\texttt{LeakyReLU}
          (spatial $128 \to 64 \to 32 \to 16 \to 8 \to 4$; каналы
          $3 \to 32 \to 64 \to 128 \to 256 \to 512$), латент~128.
          Декодер симметричен, завершается \texttt{Sigmoid}.
          Score $= q_{0{,}95}((x - \hat x)^2)$~--- 95-й перцентиль
          per-pixel MSE реконструкции (устойчивее к локальным
          артефактам, чем среднее).
    \item \textbf{GMM.} Смесь из 3 гауссиан (полная ковариация) на
          21-мерном handcrafted-векторе (см.\ табл.~\ref{tab:hybrid_clf_features}:
          вектор в точности совпадает с MLP-входом HybridLeafClassifier).
          Score $= -\log p(x)$.
    \item \textbf{OC-SVM.} \texttt{OneClassSVM} из scikit-learn с
          rbf-ядром, $\nu = 0{,}1$, признаки~--- тот же 21-мерный вектор,
          стандартизованный. Score $= -\operatorname{decision\_function}(x)$.
\end{itemize}

\subsubsection*{Данные и разбиение}

Выборка~--- та же, что у HybridLeafClassifier (ветвевые листья уровня~0).
Фиксированный seed~$=42$. Хорошие делятся 60/20/20; аномальные
(\texttt{bad\_leaf}, \texttt{non\_leaf}) распределяются поровну
между val и test (в обучении one-class не используются, поэтому
train-часть для них не выделяется).

\textbf{Обучение моделей требует только хороших листьев.}

Порог F1-max каждой модели выбирается на val (284 объекта:

percentile-подход (например, порог~--- 95-й перцентиль score'ов
на train good); он не требует bad-разметки, но даёт метрики ниже,
чем F1-max. В настоящей работе оба варианта не сравнивались~---
значения порога подобраны по F1-max для получения верхней границы
качества моделей.

    \item \textbf{GMM и OC-SVM~--- лучшие на test} (F1 $\approx 0{,}98$,

          устойчивы при $\sim 200$ train-примерах.
    \item \textbf{AE даёт низкое качество.} Test precision $= 0{,}754$~---
          около четверти хороших листьев отвергается как аномальные.
          Reconstruction-based score (per-pixel MSE) слабо разделяет
          хорошие листья и аномалии: AE достаточно хорошо реконструирует
          широкий круг RGB-кропов, в том числе бракованные листья и
          посторонние объекты, поэтому абсолютная ошибка реконструкции
          не является надёжным индикатором аномальности в данной задаче.
    \item \textbf{Сопоставление с HybridLeafClassifier.}
          GMM и HybridLeafClassifier обучаются на одном и том же
          21-мерном векторе, но с разной супервизией: GMM видит только
          хорошие листья, HybridLeafClassifier~--- все три класса.
          На бинарной задаче (good vs not-good) оба дают близкие
          метрики (F1 $\approx 0{,}979$ на test); HybridLeafClassifier
          дополнительно различает \texttt{bad\_leaf} и \texttt{non\_leaf},
          что важно для приложения, где эти два класса обрабатываются
          по-разному.
\end{enumerate}


\subsection{Поддержка типов листьев мха Sphagnum}
\label{sec:leaf_types}

Биологический объект исследования~--- листья рода \textit{Sphagnum}~---
представлен в датасете \texttt{AI\_recog\_sph} четырьмя морфологическими
группами, различающимися сложностью автоматической обработки:

\begin{itemize}
    \item \textbf{Уровень~0~--- ветвевые листья} (\textit{branch leaves},
          \texttt{lvl\_0}). Самая простая и многочисленная группа:
          одиночные листья с гладким контуром, низкая частота перекрытий.
    \item \textbf{Уровень~1~--- стеблевые листья группы
          \textit{capillifolium}} (\texttt{lvl\_1\_capillifolium\_gr}).
          Умеренная сложность.
    \item \textbf{Уровень~2~--- стеблевые листья группы
          \textit{girgensohnii}} (\texttt{lvl\_2\_girgensohnii\_gr}).
          Более сложная форма, частые перекрытия.
    \item \textbf{Уровень~3~--- стеблевые листья группы
          \textit{fimbriatum}} (\texttt{lvl\_3\_fimbriatum\_gr}).
          Наиболее сложная группа: рваные края, бахромчатая структура.
\end{itemize}

Для каждого типа листа обучается \textit{отдельный} экземпляр
HybridLeafClassifier на соответствующей подвыборке~--- это
объясняется принципиально разной формой и типичными дефектами
листьев разных групп. На момент написания работы обучение
проведено для ветвевых листьев (уровень~0), для которых собрана
наиболее представительная размеченная выборка. Обучение моделей
для уровней~1--3 планируется после пополнения разметки; архитектура
классификатора и процедура обучения при этом не изменяются,
меняется только обучающий набор и сохранённые веса.

Для типов листьев, по которым модель ещё не обучена, автоматический
режим недоступен, и пользователь работает в интерактивном режиме
(раздел~\ref{sec:solution}). Таким образом, приложение остаётся
применимым к любому типу листа независимо от текущего состояния
обученных моделей, а постепенное добавление новых классификаторов
расширяет долю снимков, обрабатываемых в полностью автоматическом
режиме.

\subsection{Тестирование}

Для верификации модуля измерений написаны модульные тесты:

\begin{itemize}
    \item \textit{Прямоугольная маска}: заданной ширины и высоты~---
          ожидаемые длина и ширина совпадают с расчётными с погрешностью менее 1\%.
    \item \textit{Эллиптическая маска}: проверяется симметричность профиля
          (ширины в точках $t$ и $1-t$ должны совпадать).
    \item \textit{Интеграционный тест}: загрузка реальных тестовых изображений из директории \texttt{test/images/},
          получение маски через \texttt{Predictor}, запуск \texttt{measure\_leaf()} и
          проверка формата вывода CSV.
\end{itemize}
